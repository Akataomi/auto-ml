# 🤖 Expert AutoML System
**Тема проекта:** Разработка экспертной системы AutoML

---

## 🎯 Мотивация и идея

Мы разрабатываем сервис, который помогает пользователю построить ML-решение **без ручной настройки всех этапов пайплайна**.

**Как это выглядит для пользователя:**
1. Пользователь загружает датасет
2. Сервис выполняет первичный анализ (EDA-lite)
3. Предлагает варианты дальнейших действий:
   - выбор постановки задачи (классификация / регрессия / временные ряды)
   - выбор моделей / архитектур (в т.ч. нейросети при необходимости)
   - базлайн и улучшения
   - оптимизация гиперпараметров
4. Проводит пользователя **по шагам**, позволяя вносить коррективы
5. Учитывает приоритет: **качество** vs **скорость**

Дополнительно рассматриваем расширение на **многомерные временные ряды** (несколько переменных).

---

## 🧩 Цели проекта

- Построить модульную AutoML-систему с понятной архитектурой и возможностью расширения
- Реализовать “экспертный режим” (советы и предлагаемые решения на каждом шаге)
- Сделать удобный интерфейс взаимодействия с пользователем (рекомендуется Streamlit)

---

## ✅ Задачи

### 1) Модульная архитектура
Разработка системы с модульной структурой (ориентир — подходы open-source библиотек), включая компоненты:
- загрузка данных
- предобработка
- генерация признаков
- выбор моделей
- оптимизация гиперпараметров
- оценка результатов
- UI слой (сервис)

### 2) Предобработка данных
- обработка пропусков
- кодирование категориальных признаков
- масштабирование числовых признаков
- обработка выбросов

### 3) Поддержка ML-задач и алгоритмов
- классификация
- регрессия
- прогнозирование временных рядов (опционально: многомерные)

### 4) Подбор гиперпараметров
- реализация механизма HPO (Grid/Random/Optuna и т.п.)

### 5) Метрики и оценка качества
- для классификации: Accuracy, Precision, Recall, F1-score, AUC-ROC
- для регрессии: RMSE, MAE, R²
- для временных рядов: RMSE/MAE/MAPE (по необходимости)

### 6) Разработка сервиса
Интерфейс взаимодействия с пользователем может быть реализован на основе:
- **Streamlit (рекомендуется)**
- tg-бот
- Flask и др.

### 7) Тестирование и сравнение
Проверка работы системы на реальных данных (например, банковская сфера / ритейл)  
и сравнение с “ручным” построением моделей.

## 🗺️ План работы (roadmap)

### Этап 1. Аналитика и постановка задачи
- формализация требований
- выбор поддерживаемых типов задач и сценариев
- дизайн архитектуры модулей
- подбор источников данных для тестирования

### Этап 2. Базовый пайплайн данных
- загрузка датасета и валидация
- EDA-lite отчёт (тип признаков, пропуски, баланс, базовые статистики)
- базовая предобработка (missing/categorical/scaling/outliers)

### Этап 3. Бейзлайны и оценка
- бейзлайн модели для классификации/регрессии
- единый интерфейс метрик и сравнения моделей

### Этап 4. AutoML: выбор модели
- автоматический выбор набора кандидатов
- оптимизация гиперпараметров
- логирование экспериментов

### Этап 5. Интерфейс сервиса
- Streamlit-приложение (загрузка данных → шаги → результаты)
- возможность ручных корректировок пользователем

### Этап 6. Временные ряды (опционально)
- поддержка TS-задач (univariate/multivariate)
- сравнение подходов и метрик

### Этап 7. Тестирование и финализация
- тесты на реальных данных (bank/retail)
- сравнение с ручным ML-пайплайном
- финальный отчёт + презентация
